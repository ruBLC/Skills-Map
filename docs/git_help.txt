краткий обзор основных команд Git, которые часто используются в контексте работы с GitHub:

git init: Инициализация нового репозитория Git в текущей директории.

git clone <url>: Клонирование репозитория с удаленного сервера на локальную машину. <url> — это адрес репозитория (например, ссылка на GitHub).

git status: Показывает статус изменений в рабочей директории.

git add <filename> или git add .: Добавление изменений в индекс. <filename> — это имя файла, которое вы хотите добавить. . добавит все файлы.

git commit -m "Описание коммита": Фиксация изменений в репозитории с комментарием.

git push <remote> <branch>: Отправка локальных коммитов на удаленный репозиторий. Обычно <remote> это origin (стандартное имя, задаваемое при клонировании), а <branch> — это имя ветки, например, master или main.

git pull <remote> <branch>: Получение изменений с удаленного репозитория и слияние их с текущей веткой.

git branch: Просмотр доступных веток. С активной веткой будет звездочка.

git checkout <branch>: Переключение на другую ветку или коммит.

git branch -b <new-branch>: Создание новой ветки и переключение на неё.

git merge <branch>: Слияние одной ветки с другой.

git log: Просмотр истории коммитов.

git remote add <name> <url>: Добавление нового удаленного репозитория.

git fetch <remote>: Загрузка изменений с удаленного репозитория без их слияния.

git diff: Просмотр различий между вашими изменениями и последним коммитом.
